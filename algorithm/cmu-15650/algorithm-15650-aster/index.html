CMU 15650 - Algorithm and Data Structure

A star Algorithm

Using a heuristic for shortest paths

Dijkstra’s algorithm assumes it knows nothing about nodes it hasn’t reached during the algorithm.
Dijkstra算法假设对没有访问过的点是一无所知的。

Suppose instead we have h(u)h(u)h(u) which is an estimate of the distance from node uuu to ttt.
现在我们假设对未访问过的点也有一定的认知，记 h(u)h(u)h(u) 是点 uuu 到 ttt 的距离。 ttt 是终点。

A* algorithm

Maintain two values for every visited node:

  • g(u)g(u)g(u) = best distance from sss to uuu found so far.
  • f(u)=g(u)+h(u)f(u)=g(u)+h(u)f(u)=g(u)+h(u) = estimate of the length of the best path from sss to ttt through uuu.
  • g(u)g(u)g(u) 是 sss 到 uuu 的距离，h(u)h(u)h(u) 是 uuu 到 ttt 的距离，加起来就是 f(u)f(u)f(u) ，也就是经过点 uuu 的 sss 到 ttt 的最短距离。

Idea: Run a Dijkstra-like algorithm using f(u)f(u)f(u) as the key
Dijkstra’s algorithm: f′(u) = d(v) + length(v,u)
其实 Dijkstra 算法只用了 g(u)g(u)g(u) , 而 h(u)=0h(u) = 0h(u)=0

Implementation

Choice of h(u)h(u)h(u)

Definition (Admissible):
Let h^\*(u) be the real shortest distance from uuu to ttt.
h^\*(u) 是从点 uuu 到 ttt 的真实的最短距离

A heuristic h(u)h(u)h(u) is admissible if h(u) ≤ h^\*(u) for all uuu.
在算法计算的过程中，只有当 h(u) ≤ h^\*(u) 时，算法才是可接受的
