Dijkstra Algorithm

Shortest path

Shortest Paths:
Given directed graph GGG with nnn nodes, and non-negative lengths on each edge, find the nnn shortest paths from a given node sss to each viv_iv​i​​.（Every viv_iv​i​​ has its own shoertest path from sss to it.）

给定一个图 GGG 和一个起点 sss ，找到从起点 sss 到余下所有其他点的最短路径。

nextEdge for Shortest Path

We discussed tree growing in the Graph Traversal
树的增长的算法

Dijkstra’s algorithm is just a special case of tree growing.

nextEdge for Shortest Path （函数nextEdge）:

  • Let uuu be some node that we have already visited (it will be in S).
    假设点uuu是我们已经访问过的一个点。我们把这些访问过的点的集合设为SSS
  • Let d(u)d(u)d(u) be the length of the s−u path found for node u∈S.
    我们记 d(u)d(u)d(u) 为从起点 sss 到点 uuu 的路径的距离
  • nextEdge: return the frontier edge (u,v)(u,v)(u,v) for which d(u)+length(u,v)d(u)+length(u,v)d(u)+length(u,v) is minimized.
    nextEdge这个函数呢，就是从集合 sss 中的点的邻点中找到一个点 uuu ，使得 d(u)+length(u,v)d(u)+length(u,v)d(u)+length(u,v) 最小
  • The “d(u)d(u)d(u)” term is the difference from Prim’s algorithm.

Dijkstra’s Algorithm (in KT book)

Algorithm

while loop: uuu 是 heap 中的一个点

For loop: 遍历 uuu 的所有邻点 vvv，这个vvv 可能是之前遍历到过的，因为 vvv 可能同时是不同 uuu 的邻点. For loop 结束以后要把 uuu 从堆中删去，因为它的历史使命完成了，以它为跳板的路线的长度都已经确定了。

Proof of Correctness

Theorem.
Let T be the set of nodes explored at some point during the algorithm. For each u ∈ T , the path to uuu found by Dijkstra’s algorithm is the shortest.
对于已经用 Dijkstra 算法访问过的任意点 uuu ，从起点 sss 到点 uuu 的距离都是最短的。

Proof:
By induction on the size of TTT . 用数学归纳法来证明

  1. Base case: When ∣T∣=1|T| = 1∣T∣=1, the only node in TTT is sss, for which we’ve obviously found the shortest path.
    第一步，如果整个图只有一个起点 sss ，那么显然我们可以得到最短路。
  2. Induction Hypothesis: Assume theorem is true when |T| ≤ k. Let vvv be the (k+1)st(k+1)^{st}(k+1)​st​​ node added using edge(u,v)(u,v)(u,v).
    归纳的假设：假设我们的定理在前 kkk 个点都是对的，我们记 vvv 是第 (k+1)(k+1)(k+1) 个点
  3. Let PvP_vP​v​​ be the path chosen by Dijkstra’s to vvv and let PPP be any other path from sss to vvv.
    我们记 PvP_vP​v​​ 为用 dijkstra 算法找到的路径， PPP 是任意一条别的路径

The path to vvv chosen by Dijkstra’s is of length ≤ the alternative blue path.
PS: Dijkstra’s choose vvv as the next node so d(y)d(y)d(y) must larger than d(v)d(v)d(v)

如上图，根据 dijkstra 算法， d(y)d(y)d(y) 都比 d(v)d(v)d(v) 要小了，不用说 d(y)+length(y,v)d(y)+length(y,v)d(y)+length(y,v) 了

Runtime of Dijkstra’s Algorithm

Same as Prim’s MST algorithm:

  1. Every edge is processed in the for loop at most once.
    This is because the graph is directed
    While 循环会遍历到每一个点的每一条边，因为边是有向的，我们只挑从 SSS 出发的边，所以是每条边一次而不是两次
  2. In response to that processing, we may either
    遍历到这条边的时候，有下面这些选择：
    (1. do nothing, O(1)O(1)O(1); —— 不满足更新条件
    (2. insert a item into the heap of at most ∣V∣|V|∣V∣ items, O(log∣V∣)O(log|V|)O(log∣V∣); —— 这是一个新的点，加到 heap 中
    (3. reduce the key of an item in a heap of at most ∣V∣|V|∣V∣ items, O(log∣V∣)O(log|V|)O(log∣V∣); —— 点已经在 heap 中了，但是有更短的路径，所以我们要更新，然后 reduce key

Total time is therefore: O(∣E∣log∣V∣)O(|E|log|V|)O(∣E∣log∣V∣).
