动态规划算法之
Sequence Alignment 序列对比

Problem

相关背景定义：

  1. 给定两个字符串
$ X = x_1 x_2 .. x+m \\ Y = y_1 y_2 .. y+n $
  1. 两个表示两个字符串不同位置的集合
    \\{1，2，3, ..., m\\} \\\\ \\{1，2，3, ..., n\\}
  2. MMM 是一个有序的配对（pair）的集合
    MMM中不能有交叉的情况出现： 所以：如果(i,j),(i′,j′)∈M(i,j),(i', j') \in M(i,j),(i​′​​,j​′​​)∈M 而且 i<i′i < i'i<i​′​​ 那么j<j′j< j'j<j​′​​

举个例子

stop-
-tops

alignment =\\{ (2,1),(3,2),(4,3) \\}

接下来就是序列对比的定义了：

  1. δ\deltaδ 代表 gap penalty, 也就是说如果要在一个string中加一个短横，也就是所谓的gap才能去和另一个string对齐，那么这个时候就要有一个gap penalty。
  2. α\alphaα 代表两个字符串p,qp, qp,q,在相同位置上的字母的不匹配的cost
    对于每一个(i,j)∈M(i,j) \in M(i,j)∈M 都会有一个 cost $\alpha_{x_i y_i}$
    如果i,ji, ji,j 位置上的两个字母相等，那么这个cost为零。
    如果αpq=0\alpha_{pq} = 0α​pq​​=0那么表示两个字符串再每一个相应的位置上的字母都是一样的。
  3. MMM这个集合的总代价就是 δ\deltaδ 和 α\alphaα 的和

然后我们的问题就是如何排列两个字符串来使得这个总代价最小
δ\deltaδ 和 α\alphaα 是外部的参数。
有一个条件就是δ+αpq<3δ\delta + \alpha_{pq} < 3 \deltaδ+α​pq​​<3δ

Design the algorithm

继续用动态规划的思想来解决问题

把问题一分为二，判断两个字符串的最后一个位置(m,n)(m,n)(m,n)是不是一个配对

  1. (m,n)∈M(m,n) \in M(m,n)∈M
  2. (m,n)∉M(m,n) \notin M(m,n)∉M

引申出一个定理：
如果(m,n)∉M(m,n) \notin M(m,n)∉M，那么XXX字符串中的的第mmm个位置和YYY字符串中的第nnn个位置肯定有一个是没有对应的配对的字母的。
证明：
设i<mi< mi<m, j<nj < nj<n， (m,j)∈M(m,j) \in M(m,j)∈M，(i,n)∈M(i,n) \in M(i,n)∈M
这个时候就出现了交叉的配对(m,j)(m,j)(m,j)和(i,n)(i,n)(i,n)是交叉的。

那么我们就可以把问题一分为三：

  1. (m,n)∈M(m,n) \in M(m,n)∈M
  2. XXX字符串中的的第mmm个位置没有对应的字母（这个时候XXX比较长）
  3. YYY字符串中的的第nnn个位置没有对应的字母（这个时候YYY比较长）

设opt(i,j)opt(i,j)opt(i,j)为问题的最小的cost。

  1. 对应第一种情况
    opt(m,n)=αxmyn+opt(m−1,n−1)opt(m,n) = \alpha_{x_m y_n} + opt(m-1,n-1)opt(m,n)=α​x​m​​y​n​​​​+opt(m−1,n−1)
  2. 对应第二种情况，我们在XXX的mmm位置上要付出一个gap penalty，δ\deltaδ
    opt(m,n)=δ+opt(m−1,n)opt(m,n) = \delta + opt(m-1,n)opt(m,n)=δ+opt(m−1,n)
  3. 对应第三种情况，我们在YYY的nnn位置上要付出一个gap penalty，δ\deltaδ
    opt(m,n)=δ+opt(m,n−1)opt(m,n) = \delta + opt(m,n-1)opt(m,n)=δ+opt(m,n−1)

根据以上的三种情况就可以得到这个问题的状态转移方程了
序列对比的最小costs，对于i⩾1i \geqslant 1i⩾1 and j⩾1j \geqslant 1j⩾1，有：

opt(i,j) = min \\{ \alpha_{x_i y_j} + opt(i-1,j-1), \delta + opt(i-1,j), \delta + opt(i,j-1)\\}

写的好看一点儿

OPT(i,j) = min \left\\{ \begin{matrix} \alpha_{x_i y_j} + opt(i-1,j-1) \\\\ \delta + opt(i-1,j) \\\\ \delta + opt(i,j-1) \end{matrix} \right .

Running Time:
一共有O(mn)个子问题，每个子问题需要O(1)，所以一共是O(mn)

然后从右往左回溯可以找到路径
