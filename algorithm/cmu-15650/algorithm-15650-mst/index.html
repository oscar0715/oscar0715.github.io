Minimum Spanning Tree

Graph

Graph: graphs specify pairwise relationship between objects.
图展示了对象与对象之间的关系

undirected tree: G=(V,E)G = (V, E)G=(V,E)
V is a set of nodes (verticles) 顶点的集合
E is a set of edges. 边的集合

Trees

Trees: A graph GGG is a tree if it is connected and contains no cycles.
如果一个图中的点都相连而且这个途中不存在环，那么这个图就是一颗树

  1. TTT is a tree
  2. TTT contains no cycles and n−1n-1n−1 edges 树有n−1n-1n−1条边
  3. TTT is connected and revoming any edge will disconnect it.
    从一棵树里移除任何一条边都会将这个树断开，因为没有环嘛。
  4. Any two nodes in a tree has only one path.
    树中的任意两个顶点间只有一条唯一的路径。
  5. TTT is acyclic and adding any new edge creates exactly one cycle.
    在树种添加任意一条边都会形成一个环。

The Minimum Spanning Tree

Given a undirected graph G with all non-negative weights.
Find a Subgraph TTT that connects all verticles and minimizes cost(T).
给定一个无向图，图中的每一条边都有一个非负的权重，要求在图中找到一个子图，这个子图连接了图中所有的点，并且要使得这个子图中的边的权重和最小。

cost(T)=∑(u,v)∈Td(u,v)cost(T)= \sum_{ (u,v) \in T}d(u, v) cost(T)=​(u,v)∈T​∑​​d(u,v)

T will be a tree:
If there is a cycle, we could remove any of the edge on the cycle to get a new subgraph T' with smaller cost(T')
得到的这个子图肯定是一棵树，否则的话，我们可以移除任意一条环中的边，这样图中的所有顶点还是相连的。

MST Property Summary

MST Cut Property

**Theorem: **
The minimum weight edge e=(v,w)e = (v, w)e=(v,w) between |S| and |V|-|S| must be in every the MST
切分定理：把一个图的顶点分为两个部分|S|，|V|-|S|，那么在所有连接这两个部分的边里面，最短的那条边肯定属于MST。

proof:
If TTT doesn’t contain eee, Because TTT is connected, there must be path PPP between vvv and www. Assume PPP contains fff that “cross the cut” between |S| and |V|-|S|.
The subgraph $ T’ = T - f \cup e$ has lower weight than TTT.
证明：如果最短边eee不在MST中，去掉连接这两个部分的边fff，再把eee加上去总是能使这棵树的总权重更小

MST Cycle Property

Theorem:
Let CCC be a cycle in GGG. Let e=(u,v)e = (u, v)e=(u,v) be the edge with maximum weight on CCC. Then eee is not in any MST of G.
环定理：如果图中有一个环，那么这个环中的最长边一定不是MST中的一条边

Proof:
Suppose the theorem is false. Let TTT be a MST that contains eee
Deleting eee from TTT, partioning TTT in to two sets: SSS(contains uuu ) and V−SV-SV−S (contains vvv).
Cycle CCC must have some other edge fff that goes from ∣S∣|S|∣S∣ and ∣V−S∣|V-S|∣V−S∣.
Replcing eee by fff produces a lower cost tree, contradicting that T is an MST.
证明：假设环中的最长边eee在MST中，∣S∣|S|∣S∣和∣V−S∣|V-S|∣V−S∣由eee相连，我们去掉eee，总能找到比eee更短的另一条边，重新把这棵树连起来，这样总权重就减小了。

Starting with any root node, add the frontieredgefrontier edgefrontieredge with the smallest weight.

If it doesn’t true, we can add a very small value ϵe\epsilon_eϵ​e​​ to the weight of every edge eee

    The pair (VT,VG−VT)(V_T, V_G - V_T)(V​T​​,V​G​​−V​T​​) is a cut of GGG

Reverse-Delete Algorithm

Algorithm:
Remov edges in decreasing order of weight, skipping those whose removal removal will disconnect the graph. (largest edge in an cycle)
每次都移除最长边，跳过那些会disconnect the graph的边

Guarantee that the graph is not disconnected, deleting edge with maximum weight eee and there must be another path between u and v

Kruskal’s Algorithm

Algorithm

Algorithm:
add edges in increasing weight, skipping those whose addition could create a cycle.
Kruskal算法，将边按从小到大的顺序添加到图里，跳过那些会形成环的边。

Proof:
(Cut Property)
SSS = nodes which vvv has a path just before eee is added
uuu is in V−SV-SV−S (otherwise there would be a cycle)
仍然可以用切分定理证明

Data Structure for Kruskal Algorithm:

接下来要解决的是用什么数据结构来构造Kruskal算法
How to check if adding an edge (u,v)(u, v)(u,v) would create a cycle?

  • If u,vu, vu,v are already in the same component.
  • We start with a component for each node.
  • Components merge when we add an edge.
  • Need a way to check uuu and vvv are in same component and to merge two components into one.

Array based Union-Find

先来介绍以数组为基础的Union-Find数据结构。

Operations

  • MakeUnionFind: create the data structure containing |S| sets, each containing one item. O(|S|)
    初始化，在一开始的时候才会用到。初始化简历|S|个集合，也就是每个顶点都自成一格集合。
  • Find(i): return the name of the set containing item iii. O(1)
    查找操作，找到某个顶点的所属的集合的名字
  • Union(a,b): merge two sets with name aaa and bbb into single set
    prepend smaller list to larger list. Set size[y] = size[y] + size[x]
    合并操作，把两个集合和并起来。总是把小的集合接到大的集合后面。

Data Structure

要用到两个数组和一个list的数组

  • UF Sizes (array) 一个数组来存每个集合的大小
  • UF Sets (array) 一个数组来存每个集合的名字
  • UF items (array of list) 每个集合都有一个元素列表，这些所有的列表组成一个集合

Runtime of Union Find (array based):

Theorem:
Any sequence of kkk union operationon a collection of nnn items takes time at most proportion to klog(k)klog(k)klog(k).
kkk次union操作的时间复杂度是klog(k)klog(k)klog(k)

Proof:
After kkk unions, at most 2k2k2k items have been involved in a union. (each union can touch at most 2 new items)

  1. Every time set(v)set(v)set(v) changes, the size of the set at least doubles.
  2. Set[v]Set[v]Set[v] can have changed at most log2(2k)timeslog_{2}(2k)timeslog​2​​(2k)times

At most 2k items have been modified at all, and each was updated at most $ log_{2}(2k) $.

So totally $ 2klog_{2}(2k) $

证明：

  1. 在kkk次union操作以后，最多涉及2k2k2k个元素，因为每次的union最多涉及两个新的元素。
  2. 每次一个顶点的所属的集合改变的时候，也就是被合并的时候，合并后的集合的大小至少比原来的集合大一倍，因为我们在合并两个集合的时候，改变元素数量小的集合的名字，保留元素数量大的集合的名字。

综上所述，在kkk次union操作以后，一个集合里最多有2k2k2k个元素，集合中的每个元素所属的集合最多可能改变了log2(2k)log_{2}(2k)log​2​​(2k)次。

所以总的时间复杂度是2klog2(2k)=klog(k)2klog_{2}(2k) = klog(k)2klog​2​​(2k)=klog(k)
