Subset Sum Problem

Problem (Subset Sum)
Given:

  1. an integer bound WWW ,
  2. and a collection of nnn items,
  3. each item with a positive, integer weight wiw_iw​i​​,

find a subset S of items to:

maximize ∑i∈Swi\sum_{i\in S} w_i∑​i∈S​​w​i​​, while keeping

∑i∈Swi⩽W\sum_{i\in S} w_i \leqslant W∑​i∈S​​w​i​​⩽W

说人话就是：

  1. 有一个数字集合 SSS 和一个边界值 WWW
  2. 在 SSS 中挑出一些数字，使得这些数字的和尽可能地接近于 WWW

Solution

Value of the OPT

Suppose for now we’re not interested in the actual set of items.
Only interested in the value of a solution
如果我们不在意这个数字集合具体有哪些数字，只是在意最终的这个接近于 WWW 的和

Optimal Notation

Notation：

  1. Let S′S'S​′​​ be an optimal choice of items
    ( S′S'S​′​​ 为最优解的集合)
  2. Let OPT(n,W)OPT(n,W)OPT(n,W) be the value of the optimal solution.
    (OPT(n,W)OPT(n,W)OPT(n,W)为最优解的集合中所有数字的和)

Subproblems:

  1. To compute OPT(n,W)OPT(n,W)OPT(n,W) : We need the optimal value for subproblems consisting of the first jjj items for every knapsack size 0⩽w⩽W0 \leqslant w \leqslant W0⩽w⩽W.
  2. Denote the optimal value of these subproblems by OPT(j,w)OPT(j,w)OPT(j,w)

为了求解 OPT(n,W)OPT(n,W)OPT(n,W) ，我们需要用到包含前 jjj 个item的子最优解 OPT(j,w)OPT(j,w)OPT(j,w)

Recurrence

状态转移公式:

OPT(j,w) = max \left\\{ \begin{matrix} OPT(j − 1, w), & j \notin S' \\\\ w_j + OPT(j − 1, w − w_j), & j \in S' \end{matrix} \right .

$ OPT(0,W) = 0 $ If no items
$ OPT(j,0) = 0 $ If no space

  1. 第一种情况，第 jjj 个 item 不在这个 S′S'S​′​​ 集合中，那么
    OPT(j,W)=OPT(j−1,W)
  2. 第二种情况，第 jjj 个 item 在这个 S′S'S​′​​ 集合中，那么
    OPT(j,W) = w_j +OPT(j−1,W−w_j)
  3. 特殊情况，第 jjj 个 item 的$ w_j > W$，那么这个 item 必然不在这个 S′S'S​′​​ 集合中，那么
    OPT(j,W)=OPT(j−1,W)

其实动态规划的关键，就是在当前这一步的时候，对这一步的情况做分类讨论。

Runtime

  1. O(nW)O(nW)O(nW) cells in the matrix.
  2. Each cell takes O(1)O(1)O(1) time to fill. （比较两个值就好了）
  3. O(n)O(n)O(n) time to follow the path backwards.
  4. Total running time is O(nW+n)=O(nW)O(nW + n)=O(nW)O(nW+n)=O(nW).

This is pseudo-polynomial because it depends on the size of the input numbers.

Knapsack 背包问题

Problem (Knapsack)

Given:

  1. an integer bound WWW ,
  2. and a collection of nnn items,
  3. each item with a positive , integer weight wiw_iw​i​​,
  4. a value vivivi for each weight

find a subset S of items to:

maximize ∑i∈Svi\sum_{i\in S} v_i∑​i∈S​​v​i​​, while keeping

∑i∈Swi⩽W\sum_{i\in S} w_i \leqslant W∑​i∈S​​w​i​​⩽W

Difference from Subset Sum: want to maximize value instead of weight.

Gready Solution

Idea:
Sort the items by pi=vi/wip_i = v_i /w_ip​i​​=v​i​​/w​i​​ 按单价排序
Larger viv_iv​i​​ is better, smaller wi is better.
If you were allowed to chose fractions of items, this would work.

This greedy algorithm doesn’t work for knapsack where we can’t take part of an item. 但是如果不能拆，单价排序可能不管用了。

举个例子，四件物品

  1. w=1w = 1w=1, v=30v = 30v=30, v/w=30v / w = 30v/w=30
  2. w=2w = 2w=2, v=40v = 40v=40, v/w=20v / w = 20v/w=20
  3. w=3w = 3w=3, v=45v = 45v=45, v/w=15v / w = 15v/w=15
  4. w=4w = 4w=4, v=100v = 100v=100, v/w=25v / w = 25v/w=25
  5. 限制条件 W=6W = 6W=6

能拆的话选择1, 4 和一半的2
V=30+100+40/2=150V = 30 + 100 + 40/2 = 150V=30+100+40/2=150

不能拆的话，只能选1，4
V=30+100=130V = 30 + 100 = 130V=30+100=130
但是更好的选择是2，4
V=40+100=140V = 40 + 100 = 140V=40+100=140
