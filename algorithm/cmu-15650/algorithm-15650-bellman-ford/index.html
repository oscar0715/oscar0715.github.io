CMU 15650 - Bellman-Ford

Post modified: 2017-10-15 | In Technology , Algorithm

CMU 15650 - Algorithm and Data Structure

Bellman-Ford Algorithm

Shortest Path Problem

Shortest Path with Negative Weights:
Given directed graph GGG with weighted edges d(u,v)d(u,v)d(u,v) that may be positive or negative, find the shortest path from sss to ttt.
给定有向有权图，边的权重有正有负，找到 sss 到 ttt 的最短路径

Complication of Negative Weights

Negative cycles:
If some cycle has a negative total cost, we can make the s − t path as low cost as we want:
如果在图中存在一个总权重为负数的环，那么我们可以使包含这个环的路径的总长度到负无穷

Go from sss to some node on the cycle, and then travel around the cycle many times, eventually leaving to go to ttt.
因为我们可以一直沿着这个负环走啊走啊走啊

Bellman-Ford

  1. Let dists(v)dists(v)dists(v) be the current estimated distance from sss to vvv.
    我们假设 dists(v)dists(v)dists(v) 是从点 sss 到 vvv 的估计的距离
  2. At the start, dists(s)=0dists(s) = 0dists(s)=0 and dists(v)=∞dists(v) = \inftydists(v)=∞ for all other vvv.
    起点的 dists(s) = 0$，其他点在初始情况下都是无穷大
  3. Ford step.
    Find an edge(u,v)edge(u,v)edge(u,v) such that
    dists(u)+d(u,v)<dists(v)dists(u) + d(u,v) < dists(v)dists(u)+d(u,v)<dists(v)
    and set dists(v)=dists(u)+d(u,v)dists(v) = dists(u) + d(u,v)dists(v)=dists(u)+d(u,v).
  4. Repeatedly Applying Ford Step until dists(u)+d(u,v) ≥ dists(v)

其实 Dijkstra 也是会做这一步。

Theorem:
After applying the Ford step until

dists(u)+d(u,v) ≥ dists(v)

for all edges, dists(u)dists(u)dists(u) will equal the shortest-path distance from sss to uuu for all uuu.

只要不断重复的执行 Bellman-Ford Step，最后所有的 dists(v)dists(v)dists(v) 都已经是最小的时候，就找到了最短路，因为不能更短了。

那么问题来了，我们怎么去找还可以做 Bellman-Ford Step 的边呢
Which edges are candidates for the Ford rule?
Ford rule:

dists(u)+d(u,v)<dists(v)dists(u) + d(u,v) < dists(v) dists(u)+d(u,v)<dists(v)

This can only become true if dists(u)dists(u)dists(u) has gotten smaller since the
last time we checked.
如果 dists(v)dists(v)dists(v) 需要被更新的话，原因肯定是它之前的那个节点 uuu 的 dists(u)dists(u)dists(u) 变小了

  • Whenever we change dists(u)dists(u)dists(u) we add u to a queue.
    在改变 dists(u)dists(u)dists(u) 的时候，我们就把 uuu 加到一个 queue 里面，这就是做了一个标记。
  • To look for an edge to apply the Ford rule to we take a node of the queue and look at all its edges.
    然后要做的就是从 queue 里面取出一点，然后去检查这个点的所有相邻的点就好了。

Implementation

Runtime

  • n = number of nodes
  • m = number of edges

After dists(v)dists(v)dists(v) has been updated kkk times, it corresponds to a simple path of kkk edges. （这个地方不是很理解……）（每做一次更新，都能确定一个新的节点的最短路）

A path can contain at most n − 1 edges (before nodes start to repeat), so each dists(v)dists(v)dists(v) can be updated at most n − 1 times.

Updating all vertices once takes time O(m)O(m)O(m) since we look at each edge twice.
Total running time = O(mn)O(mn)O(mn). （m就是所有点的度的总和，这个好说）

Note that this is slower than Dijkstra’s algorithm in general.

Dijkstra 算法的本质是利用了贪心思想，是在剩下的节点中，找一个和 SSS 最近的节点。
但是 Bellman Ford 算法不能这样做，因为有负边，所以只能去检查每一条边。
